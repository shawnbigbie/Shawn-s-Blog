Todo App Backend

Posted by Shawn Bigbie on Oct 28th, 2020

How did you design and architect your app and server?

I decied to change the layout and architect of my app to better fit the project also expand my skills that I wanted to test out like passport and express session.

The design I choosed was a modern and simplicity for the user not to get to confussed and feel like the design was over done or unpleasant. To do this style and layout I used bootstrap to save time remakeing the whole design and layout from my last run at the todo app project.

I added the authentication for users and for each user to create an account for privacy since I thought about if the list was something that they dident want everyone to see or even if they just wanted it to be personal they could have that benefit.

They can create an account real easy that will be stored on the database with the todo data that they add to their account.

How did the front end connect to the backend?

For the backend of the app I picked mongodb hosted on atlas with mongoose, then had my api and routes built within the app compare to fetching the todo api and hosting it separate. I also used handlebars for the views and front-end to display the data.

for the front-end communicating to the database used mongoose and modules to store the todo and user infomation.

Why did you choose the design you did?

I choose to design the app the way I did to have it be able to expand and be more then a todo list in the future. Since there is the authentication added to this and users this extends the use of this todo app not only be a list but could be infomation tables or a shopping cart if needed.

How did you model you data and why?

I decided to have the catagory be apart of the todo model instead of it being a separate model since it wouldn't been worth the while to make all of that. You could instead just add a search filter to filter throw the catagory data with out it being stored in the database as its separate data point.
